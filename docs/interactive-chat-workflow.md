# Interactive Chat Fiction Workflow with Mushni

This workflow explains how to create interactive chat fiction, character scenes, and playable story projects.

## Goal

Create a structured interactive story project with characters, scenes, branches, and publishable metadata.

## Steps

1. Define the project premise.
2. Create character cards.
3. Define the opening scene.
4. Write the conversation style and boundaries.
5. Generate scene messages.
6. Add optional branches or player choices.
7. Test the player experience.
8. Review safety, tone, and continuity.
9. Publish or revise the project.

## Recommended Mushni Path

- Use GameXAI.one for interactive chat fiction and playable story workflows.
- Use character cards to keep identity, voice, and relationship context stable.
- Use project metadata to prepare the work for browsing, sharing, and publishing.

## Quality Checklist

- The player understands the opening situation.
- Character voice remains consistent.
- Branches do not contradict the story setup.
- Generated messages are reviewed before publishing.
- The project follows platform and local content rules.
