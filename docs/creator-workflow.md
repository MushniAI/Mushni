# Creator Workflow

This document describes a practical production workflow for AI-assisted creative projects.

## Workflow Overview

```text
Idea -> Brief -> Characters -> Outline -> Scenes -> Assets -> Review -> Publish
```

## Stage 1: Idea

Write the creative idea in one paragraph. Keep it specific enough to guide the workflow, but flexible enough to improve during iteration.

## Stage 2: Brief

Convert the idea into a project brief:

- Title
- Format
- Audience
- Genre
- Tone
- Main promise
- Publishing channel
- Constraints

## Stage 3: Characters

Create character cards for every recurring character. Store stable traits in one place so future prompts can reference the same details.

Use [character-card.md](../templates/character-card.md).

## Stage 4: Outline

Create a beginning, middle, and end. For longer works, split the outline into chapters, episodes, or scenes.

Use [story-outline.md](../templates/story-outline.md).

## Stage 5: Scenes and Storyboards

Break the outline into scene-level tasks:

- Scene goal
- Characters present
- Location
- Key action
- Emotional beat
- Visual notes
- Dialogue notes

For visual projects, use [storyboard-shot-list.md](../templates/storyboard-shot-list.md).

## Stage 6: Asset Production

Generate or assemble the assets needed for the project:

- Text drafts
- Image prompts
- Character references
- Background references
- Voice or video planning notes
- Editing notes

## Stage 7: Human Review

Review every important output before use. Check quality, safety, rights, and continuity.

## Stage 8: Publish and Learn

Track what worked:

- Which prompts were stable
- Which scenes needed the most revision
- Which assets can become reusable templates
- Which audience feedback should influence the next version
