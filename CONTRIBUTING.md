# Contributing to Mushni

Thanks for your interest in improving the Mushni public resource hub.

## Good Contributions

- Creator workflow examples
- Prompt patterns that improve consistency
- Documentation improvements
- Template improvements
- Safety and compliance notes
- Clear bug reports or feature requests

## Before You Contribute

Please make sure your contribution:

- Is related to AI-assisted creative workflows
- Does not include private, copyrighted, or confidential material without permission
- Does not promote spam, deception, or unsafe content
- Is written clearly enough for other creators to reuse

## Opening Issues

When opening an issue, include:

- What you are trying to create
- Which workflow or template you used
- What did not work
- What result you expected

## Pull Requests

For pull requests:

1. Keep the change focused.
2. Use clear file names.
3. Explain why the change is useful.
4. Avoid adding unrelated promotional links.
5. Keep examples safe, lawful, and respectful.
